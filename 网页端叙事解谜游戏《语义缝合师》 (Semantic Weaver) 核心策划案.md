# 网页端叙事解谜游戏《语义缝合师》 (Semantic Weaver) 核心策划案

## 一、 游戏简介与目标平台

- **游戏名称**：《语义缝合师》 (Semantic Weaver)
- **游戏类型**：2D 纯UI交互、叙事解谜、Meta-Game（元游戏）。
- **核心体验**：通过阅读理解、提取关键词，并在推演板上进行“逻辑合成”，体验抽丝剥茧的推理快感与细思极恐的剧情反转。
- **目标平台**：纯前端 H5 网页游戏（推荐 PC 端浏览器游玩以获得最佳拖拽体验）。
- **部署环境**：静态网页，部署于腾讯云 EdgeOne。无后端数据库依赖。
- **存档机制**：使用浏览器原生的 `localStorage` 保存玩家当前到达的章节（Trial）进度。

## 二、 世界观与背景设定

- **黑色集团与记忆碎片**：在赛博朋克近未来，统治城市的“黑色集团”为了窃取受害者大脑中的绝密情报，会进行残酷的审讯。受害者的大脑为了自我保护，会将记忆“碎片化”（变成毫无关联的感官词汇，如“大雨”、“红色的液体”）。
- **主角的身份**：玩家扮演集团最尖端的“语义缝合师”（代号04）。拥有将碎片化记忆重新拼接还原的能力。
- **核心诡计（认知滤网）**：为了防止主角在审讯无辜者时产生精神崩溃和同情心，集团在主角脑内植入了【认知滤网】。
  - 主角眼中的自己：一位善良的心理医生，在温馨的地下诊所里帮助失忆的可怜人找回记忆。
  - 残酷的真相：主角其实是黑色集团最完美的“刑具”，在一个血迹斑斑的情报提取室里，用极致的“温柔”骗过受害者的潜意识防线，榨取密码。情报一旦榨出，受害者即被销毁。

## 三、 游戏具体核心玩法 (Gameplay Mechanics) - **[程序实现重点]**

游戏的底层逻辑是一个基于 JSON 字典的 **“多对一合成状态机”**。全程无动作要素，仅通过鼠标操作。

### 1. 核心操作循环

- **提取 (Extract)**：NPC 在左侧输出对话文本。文本中特定的“关键词”带有高亮 CSS 样式。玩家可以用鼠标按住这些关键词，将其拖拽（Drag）到右侧的【推演板】区域。拖拽后，该词汇在推演板上生成一个独立的实体“词汇卡片”。
- **合成 (Combine)**：玩家在右侧【推演板】上，将 [卡片A] 拖拽并重叠到 [卡片B] 上（Drop）。
  - **判定成功**：如果底层 JSON 字典中存在 `A+B` 的配方，则播放白光特效与成功音效，销毁A和B，在原位置生成一个全新的进阶词条 [卡片C]。
  - **判定失败**：配方不存在，两张卡片弹开，屏幕边缘闪烁红光，提示“缺乏逻辑关联”。
- **提交与推进 (Submit & Progress)**：玩家将合成出的【金色最终线索卡片】，从右侧反向拖回左侧的对话框区域。系统验证成功后，当前对话阶段结束，NPC 触发下一段新的剧情文本。

### 2. 给 AI 开发者的底层数据结构示例 (JSON 驱动)

> **AI 开发要求**：游戏的全部剧情推进、台词渲染、合成判定，**必须完全由一个外部的 JSON 文件配置控制**。主程序只负责读取和渲染，做到数据与逻辑完全分离。

## 四、 剧情主线与章节划分 (4个 Trial)

游戏剧情采用“单元剧铺垫，最终章收网”的结构。

- **Trial 1：失落的铁盒（教学关）**
  - **NPC**：穿着雨衣的数据快递员。
  - **剧情**：他忘记了丢失的铁盒放在哪里。玩家通过合成“大雨”、“福尔马林味”等，帮他想起了坐标。
  - **结局**：NPC微笑着感谢医生后离开（表象）。
- **Trial 2：主妇的偏执（进阶关）**
  - **NPC**：坚称丈夫是外星人的焦虑主妇。
  - **剧情**：玩家用更复杂的逻辑，解开她脑中的隐喻，发现她其实是反叛军的家属，藏着一份名单。
- **Trial 3：无脸的医生（高难度关）**
  - **NPC**：患有PTSD的退役雇佣兵。
  - **剧情**：他脑海里一直有一个拿着手术刀的黑影（其实就是主角自己）。玩家引导他还原武器图纸密码。
- **Trial 4：认知滤网（终极反转）**
  - **触发条件**：Trial 3 结束后自动触发。
  - **剧情**：UI 发生视觉崩溃。左侧不再出现病人，而是直接输出系统日志（展示前三人被提取密码后直接处决的真相）。主角发现自己才是黑手。
  - **最终玩法**：系统开始读条格式化主角的大脑。玩家必须在倒计时内，用推演板上受害者遗留的痛苦记忆碎片，合成出终极武器——**【名为真相的逻辑炸弹】**，并拖拽砸向左侧的【集团云端数据库】目标位。
  - **双结局**：
    - **结局A（反抗/救赎）**：成功合成炸弹并提交。主角烧毁自己大脑，将真相全网广播，摧毁集团。
    - **结局B（懦弱/循环）**：倒计时结束未完成，或将真相词卡扔进垃圾桶。UI 恢复温馨，主角记忆被重置，迎来下一个永远无法拯救的病人。

## 五、 游戏 UI 设计与双态切换 (Dual-State UI)

整个界面采用“系统设备”沉浸式设计，屏幕被分为左右两半。

### 1. 顶部状态栏 (HUD)

- **左侧**：平时显示常规的 `Session_ID: 014`。最终章突变为红色的 `警告：认知滤网已破碎`。
- **中央**：一个名为【解析进度】的线框进度条。平时按解谜阶段上涨；最终章突变为红色的【记忆格式化进度】，带有强烈的读条压迫感。
- **右侧**：随机跳动的伪系统数据（如 `MEM: 45%`），营造终端机氛围。

### 2. 左半屏：会诊记录区 (Dialogue Area)

- 上方为 NPC 的厚涂油画风格半身立绘。
- 下方为打字机效果的文本框（使用 Monospace 等宽字体）。
- **视觉状态 A（前三关）**：温馨的复古地下诊所背景，暖黄色调，木质边框。
- **视觉状态 B（最终章）**：背景 CSS Glitch 故障闪烁，瞬间切换为冰冷的金属审讯室背景，血迹斑斑。

### 3. 右半屏：心理推演板 (Evidence Board)

- 玩家的主操作区，支持拖拽、吸附和卡片重叠判定。
- 底部常驻三个玩家自带的技能卡片（无法被消耗）：【视觉重构】、【听觉重构】、【嗅觉重构】。
- 右下角常驻一个“垃圾桶”区域，用于销毁合成错误的废料词条。
- **视觉状态 A**：质感真实的软木板背景（Corkboard），词条呈现为带有图钉的彩色便利贴。
- **视觉状态 B**：软木板消融，变成带红色警告代码的战术全息投影网格，卡片变为冰冷的数据块。

## 六、 游戏美术与台词风格

- **美术风格**：《极乐迪斯科》式的厚涂油画风（Impasto）结合复古 CRT 显示器美学。不需要花哨的3D粒子动画，主要依赖静态高质感背景图 + CSS 霓虹发光/阴影/故障特效。
- **台词风格 (Dialogue Tone)**：
  - **NPC台词**：意识流、充满隐喻、带有极强的心理恐惧与焦虑感。（例如：“那条走廊像是一条正在消化的胃管，墙壁上渗出红色的粘液……”）
  - **系统台词（最终章）**：极致的冰冷、客观、残酷，没有一丝情感。（例如：“`> 提取完成。目标价值清零，进入物理销毁程序。`”）

## 七、 补充：给 AI 的核心开发架构建议

*(请直接参照此规范进行代码生成)*

1. **技术栈**：推荐使用纯 HTML5 + CSS3 + 原生 JavaScript (Vanilla JS)。若需组件化，可使用极轻量级的 Vue.js 3（无需构建工具，直接 CDN 引入）。
2. **拖拽 API**：使用 HTML5 原生的 `dragstart`, `dragover`, `drop` 事件实现左右区域的数据传递与卡片合成。
3. **视觉切换**：UI 的“状态A”到“状态B”的切换，必须通过在 `<body>` 标签上切换 CSS Class（例如从 `.theme-clinic` 切换为 `.theme-interrogation`）来实现，以确保全局视觉瞬间突变。
4. **动效实现**：CRT 扫描线、文字打印效果、卡片合并时的发光闪烁，尽量使用 CSS `@keyframes` 和 `box-shadow` 实现，保持代码极致轻量，确保在网页端运行丝滑无卡顿。
