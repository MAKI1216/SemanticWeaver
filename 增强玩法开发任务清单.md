## Phase 1：基础设施重构

> 目标：不改玩法，只升级底层架构，为后续所有增强机制做准备。
> 验收标准：现有游戏功能完全不变，但数据结构和卡片系统已支持新特性。

### 任务 1.1 — 数据结构升级

**文件**：`data/game-data.js`

- [ ] 为每个关键词添加 `attributes` 字段（空数组，后续 Phase 2 填充）
- [ ] 为每个关键词添加 `card_type` 字段（`"normal"` | `"hidden"` | `"half_finished"` | `"distorted"` | `"meta"`）
- [ ] 为每个关键词添加 `is_extractable` 布尔字段（控制是否可从对话中提取）
- [ ] 为每个 Stage 添加 `is_final_stage` 布尔字段（仅最终 Stage 为 true）
- [ ] 保留 `required_submit` 字段用于向后兼容
- [ ] 预留 `conclusions` 数组字段（Phase 4 填充）
- [ ] 预留 `hidden_keyword_sources` 字段（Phase 2/3 填充）

**验收**：游戏数据加载正常，所有现有 Trial 可正常游玩，控制台无报错。

### 任务 1.2 — 卡片状态系统

**文件**：`js/board.js`、`js/renderer.js`

- [ ] 卡片创建时读取 `card_type`，根据类型应用不同视觉样式：
  - `normal`：现有样式不变
  - `hidden`：显示为乱码/噪点纹理，不可拖拽（Phase 2/3 解锁后变为 normal）
  - `half_finished`：虚线边框 + 半透明（Phase 8 使用）
  - `distorted`：glitch 文字效果 + 随机色偏（Phase 2 使用）
  - `meta`：特殊金色边框 + 微弱脉动光晕（Phase 3 使用）
- [ ] 卡片状态切换函数 `setCardType(card, newType)`，自动更新视觉样式
- [ ] 卡片销毁动画区分类型（normal 溶解 / hidden 噪点消散 / distorted glitch 碎裂）

**验收**：现有 normal 类型卡片表现完全不变，其他类型卡片有正确的视觉样式预留。

### 任务 1.3 — 关键词可重复提取

**文件**：`js/dialogue.js`、`js/board.js`

- [ ] 移除关键词提取后的"已提取"标记逻辑（提取后对话文本中关键词仍高亮可交互）
- [ ] 同一关键词可多次提取，每次提取生成一张新的卡片实例
- [ ] 对话文本中关键词的高亮样式不变（{关键词}花括号语法保持）
- [ ] 提取时播放音效，但第二次提取同一词时音效更轻（避免重复提取刷音效）

**验收**：玩家可以从对话中反复提取同一关键词，合成失败后原料消失但可以重新提取。

### 任务 1.4 — 推演板重置功能

**文件**：`js/board.js`、`js/renderer.js`、`css/styles.css`

- [ ] 在推演板边缘添加"重置"按钮（不显眼，小号文字按钮）
- [ ] 点击后弹出非阻断确认："重新梳理推演板？已提取的关键词会保留。"
- [ ] 确认后：所有推演板上的卡片溶解动画 → 消失
- [ ] 已提取到"关键词库"（如果有）的关键词保留
- [ ] 叙事进度完全不变（当前 Stage、当前 Trial 不变）
- [ ] 添加 NPC 台词包装（随机选一句）：
  - "你深吸一口气，把推演板上的线索重新理了一遍。"
  - "你揉了揉太阳穴，决定从头梳理。"
  - "你把散落的卡片收回手中。"

**验收**：重置后推演板清空，但玩家可以立即重新提取关键词继续游戏，不丢失任何进度。

### 任务 1.5 — 存档结构升级

**文件**：`js/game.js`

- [ ] 存档结构升级为策划案中的完整格式（见真假结论系统文档 5.2 节）
- [ ] 新增字段：`trials_state`（每关状态）、`memory_echo`（残影配置）、`playthrough_history`（历史记录）
- [ ] 旧存档自动迁移（检测 version 字段，无 version 的视为 "1.0" 并自动升级）
- [ ] 存档/读档功能正常，不丢失现有进度

**验收**：旧存档可自动迁移并正常继续游戏，新存档结构完整。

## Phase 2：关键词属性系统

> 目标：引入属性标签和矛盾标记机制，让合成有决策深度。
> 验收标准：玩家可以查看关键词属性，矛盾标记可解锁隐藏关键词，错误合成产出失真卡。

### 任务 2.1 — 属性数据填充

**文件**：`data/game-data.js`

- [ ] 定义属性枚举：`emotion`(🔥) / `fact`(🧊) / `conflict`(⚡) / `memory`(🌫) / `locked`(🔒) / `sensory`(👁) / `medical`(⚕)
- [ ] 为 Trial 1-4 所有现有关键词标注 1-3 个属性
- [ ] 定义矛盾属性对：
  - `locked` 与所有其他属性矛盾
  - `fact` 与 `emotion` 矛盾（事实 vs 情感判断）
  - `medical` 与 `sensory` 矛盾（医疗术语 vs 感官描述）
- [ ] 为 Trial 1-3 真结论路径定义矛盾标记点（见真假结论文档）：
  - Trial 1：Stage 2"很重" ↔ Stage 3"小小的"（重量矛盾）
  - Trial 3：Stage 3"手术台"出现在战场闪回中（语义矛盾）

**验收**：所有关键词有属性标签，矛盾对定义完整。

### 任务 2.2 — 属性检查 UI

**文件**：`js/board.js`、`js/renderer.js`、`css/styles.css`

- [ ] 关键词卡片上显示小型属性图标（1-3 个，排列在卡片底部）
- [ ] 鼠标悬停属性图标时显示属性名称 tooltip
- [ ] 新增"属性检查"操作：长按卡片（或右键）→ 弹出属性详情面板
- [ ] 属性检查**不消耗卡片**（免费操作）
- [ ] 属性详情面板显示：属性列表、矛盾属性提示（"此关键词与 🔒封锁 属性矛盾"）

**验收**：玩家可以免费查看任何卡片的属性，了解潜在矛盾。

### 任务 2.3 — 属性兼容性判定

**文件**：`js/board.js`、`js/game.js`

- [ ] 合成判定逻辑扩展：先查配方匹配 → 再查属性兼容性
- [ ] 如果配方匹配但属性矛盾 → 合成失败，产出失真卡（Phase 2.5）
- [ ] 如果配方匹配且属性兼容 → 正常合成
- [ ] 如果配方不匹配 → 维持现有行为（无反应）
- [ ] 某些高级配方要求特定属性组合才能触发（在 game-data.js 中配置 `required_attributes`）

**验收**：属性矛盾时合成会失败并产出失真卡，属性兼容时正常合成。

### 任务 2.4 — 矛盾标记系统

**文件**：`js/board.js`、`js/renderer.js`、`js/dialogue.js`

- [ ] 当玩家通过属性检查发现两个关键词存在矛盾时，可以"标记矛盾"
- [ ] 标记操作：同时选中两张卡片 → 点击"标记矛盾"按钮
- [ ] 标记成功后：
  - 两张卡片之间出现红色虚线连接
  - 触发 `onContradictionFlagged(contradictionId)` 事件
  - 解锁对应的隐藏关键词（将 hidden 类型卡片变为 normal）
  - 解锁对话文本中被屏蔽的词（was_noise 的词变为可提取）
- [ ] 标记失败（两个词无矛盾）时：温和提示"这两个词之间似乎没有矛盾"
- [ ] 记录到存档 `contradictions_flagged` 数组

**验收**：Trial 1 中标记"很重"与"小小的"的重量矛盾后，"死信箱"和"交接点"变为可提取。

### 任务 2.5 — 失真卡机制

**文件**：`js/board.js`、`js/renderer.js`、`css/styles.css`

- [ ] 错误合成（属性矛盾）产出失真卡：`card_type = "distorted"`
- [ ] 失真卡视觉：文字 glitch 效果（随机偏移 + 色差），属性图标随机化
- [ ] 失真卡不能用于正确配方（合成时被拒绝，温和提示"这张卡似乎不太对"）
- [ ] 失真卡修复方式：
  - 用感官重构技能卡与失真卡合成 → 净化为正常卡（消耗技能卡使用次数？或免费？待定）
  - 两张失真卡合成 → 有概率互相校准出正确关键词
- [ ] 推演板上失真卡超过 3 张时：推演板边缘出现黄色"过载"警告
- [ ] 失真卡可通过推演板重置清除

**验收**：错误合成有明确反馈（失真卡），失真卡可修复，不会永久卡死玩家。

## Phase 3：Meta 语义入侵

> 目标：让关键词能拖出推演板，入侵游戏 UI，揭示隐藏信息。
> 验收标准：Meta 关键词可拖到对话框/主题按钮/CRT 区域，触发对应入侵效果。

### 任务 3.1 — 隐藏台词层系统

**文件**：`js/dialogue.js`、`js/renderer.js`、`data/game-data.js`

- [ ] 对话文本支持双层渲染：`surface_dialogue`（表面台词）+ `hidden_layer`（底层台词）
- [ ] 表面台词正常显示，底层台词默认不可见
- [ ] 在 game-data.js 中为 Trial 2 Stage 3、Trial 3 Stage 4 添加 `hidden_layer` 数据
- [ ] 底层台词显示时使用特殊样式：等宽字体、微弱绿色辉光、与表面台词形成视觉对比

**Trial 2 底层台词**：
- 表面："我姐夫去年就消失了，他们说他是被叛军……"
- 底层："目标#037已清除，关联人定位已更新"

**Trial 3 底层台词**：
- 表面："白光……手术台……不对，是战场……是战场才对"
- 底层："受试者#019，第三轮记忆剥离术后，残留协议待提取"

**验收**：默认只看到表面台词，Meta 入侵后可看到底层台词。

### 任务 3.2 — Meta 关键词类型

**文件**：`js/board.js`、`js/renderer.js`、`css/styles.css`

- [ ] Meta 关键词卡片：`card_type = "meta"`，金色边框 + 微弱脉动光晕
- [ ] Meta 关键词可拖出推演板范围（其他类型卡片拖出推演板范围会弹回）
- [ ] Meta 关键词拖出推演板时，可接收拖放的 UI 元素高亮提示
- [ ] Meta 关键词使用后消耗（一次性），不可重新提取（与普通关键词不同）
- [ ] 每个 Meta 关键词配置 `meta_targets`：可入侵的 UI 元素列表

**验收**：Meta 关键词视觉独特，可拖出推演板，使用后消失。

### 任务 3.3 — UI 拖放入侵目标

**文件**：`js/renderer.js`、`js/board.js`、`css/styles.css`

- [ ] **对话框入侵**：Meta 关键词拖到 NPC 对话框 → 显示底层台词 → 从底层台词中提取隐藏关键词
  - 入侵动画：对话框文字 glitch → 底层文字浮现 → 新关键词可提取
  - 入侵后对话框恢复正常，底层台词中的关键词已加入可提取列表
- [ ] **主题切换入侵**：Meta 关键词拖到主题切换按钮 → 强制切换主题
  - 某些隐藏关键词只在特定主题下可提取（如 theme-interrogation 下可见"被划掉的文字"）
  - 入侵消耗 Meta 关键词，主题切换有 5 秒过渡动画
- [ ] **CRT 区域入侵**：Meta 关键词拖到 CRT 扫描线区域 → 强化故障效果 → 隐藏在扫描线背后的文字浮现
  - 浮现的文字可作为可提取关键词
  - 效果持续 10 秒后恢复正常

**验收**：三种入侵目标都可接收 Meta 关键词，触发对应效果，消耗 Meta 关键词。

### 任务 3.4 — Trial 3 Meta 入侵伏笔

**文件**：`data/game-data.js`、`js/game.js`

- [ ] Trial 3 Stage 5 末尾，当玩家提交假结论后，对话中出现一个 Meta 关键词："系统漏洞"（或类似）
- [ ] 此 Meta 关键词只能拖到对话框，入侵后显示一句神秘底层台词："你有没有想过……这个界面本身也是线索？"
- [ ] 这句台词不解锁任何新关键词，纯叙事暗示
- [ ] 如果玩家提交真结论，此 Meta 关键词不出现（真结论路径已足够暗示）

**验收**：Trial 3 假结论路径出现 Meta 关键词伏笔，入侵后显示暗示台词。

### 任务 3.5 — Trial 4 Meta 入侵全面应用

**文件**：`data/game-data.js`、`js/game.js`、`js/renderer.js`

- [ ] Trial 4 中 Meta 关键词大量出现（"真实记忆"、"权限密钥"、"系统后门"等）
- [ ] Trial 4 的 Meta 入侵是获取关键合成材料的途径之一
- [ ] 但每个 Meta 入侵点都有常规备选路径（防卡关铁律）
- [ ] Trial 4 的 Meta 入侵可以触发额外剧情碎片（系统崩溃日志、被删除的记录等）

**验收**：Trial 4 Meta 入侵有实质玩法价值，但不依赖 Meta 入侵也能通关。

## Phase 4：真假结论系统

> 目标：前三关每关有真假两个结论，三关全真才能进 Trial 4。
> 验收标准：可提交真/假结论，NPC 反应分化，假结局触发正确，Trial 4 门控生效。

### 任务 4.1 — 结论数据填充

**文件**：`data/game-data.js`

- [ ] 为 Trial 1 最终 Stage 添加 `conclusions` 数组（假结论 + 真结论）
- [ ] 为 Trial 2 最终 Stage 添加 `conclusions` 数组
- [ ] 为 Trial 3 最终 Stage 添加 `conclusions` 数组
- [ ] 每个结论包含完整数据：`id`、`type`、`label`、`recipe`、`npc_reaction`（phase_1 + phase_2 + final_dialogue）、`crt_effect`
- [ ] 真结论的 `hidden_keyword_sources` 指向 Phase 2/3 实现的隐藏关键词
- [ ] 参照真假结论系统文档第一章的完整设计

**验收**：三个 Trial 的最终 Stage 各有两个可提交的结论。

### 任务 4.2 — 结论提交 UI

**文件**：`js/board.js`、`js/renderer.js`、`js/game.js`、`css/styles.css`

- [ ] 最终 Stage 的提交区改为可接收任意结论卡片（不限于一个 `required_submit`）
- [ ] 提交金色卡片后进入两步确认流程：
  - Step 1：NPC Phase 1 反应台词显示（满意/迟疑）
  - Step 2：NPC Phase 2 台词 + 确认按钮："确定提交此结论吗？"
  - 玩家可选择"确定提交"或"再想想"（退回卡片到推演板）
- [ ] 真结论提交时在 Phase 1 和 Phase 2 之间触发 CRT 异常效果
- [ ] 确认提交后：记录结论类型到存档 → 显示 final_dialogue → 进入下一关或触发结局判定

**验收**：两步确认流程完整，玩家有机会在 NPC 反应后反悔。

### 任务 4.3 — CRT 异常效果组件

**文件**：`js/renderer.js`、`js/audio.js`、`css/styles.css`

- [ ] 实现三档 CRT 异常效果（参照真假结论文档中的 CRT_EFFECTS 配置表）：
  - **Trial 1（subtle）**：扫描线抖动 800ms + 亮度降 10% + 微弱电流杂音
  - **Trial 2（moderate）**：扫描线抖动 1000ms + 偏红色温 + 0.3s 静音 + 低频嗡鸣 + NPC 瞳孔放大
  - **Trial 3（intense）**：扫描线抖动 1200ms + 偏绿色温 + 心电监护仪"嘀——" + 白噪音 + NPC 面部像素化
- [ ] 效果可复用（通过 `triggerCRTEffect(configKey)` 调用）
- [ ] 效果期间对话文字也轻微抖动

**验收**：三档 CRT 效果视觉/音频表现符合设计，不影响后续 UI 功能。

### 任务 4.4 — 结论判定与门控

**文件**：`js/game.js`

- [ ] Trial 3 完成后执行 `checkTrial4Unlock(saveData)`：
  - 三关全 true → `triggerTrial4EntryRitual()`（Phase 5 实现，此阶段先占位跳转）
  - 任意一关 false → `triggerFalseEnding(saveData)`
- [ ] 结论记录写入存档 `trials_state.trialN.conclusion_type`
- [ ] `trial4_unlocked` 派生状态正确计算
- [ ] 假结局后激活 `memory_echo.enabled = true`
- [ ] 调用 `recordMissedClues(saveData)` 记录错失的线索（为 Phase 7 准备）

**验收**：三关全真进入 Trial 4（此阶段可先简单跳转），有假结论触发假结局。

### 任务 4.5 — 假结局实现

**文件**：`js/game.js`、`js/renderer.js`、`css/styles.css`

- [ ] 假结局过场序列：
  1. 画面渐暗 + CRT 关闭效果（缩为亮线 → 消失）
  2. 黑屏 2 秒
  3. 白色文字打字机效果，每句间隔 1.5 秒
  4. 动态反问句：根据哪几关为假，只显示对应关卡的反问句
  5. "算了。也许只是错觉吧。"
  6. 系统提示（等宽字体，绿色）："记忆归档完成。诊疗周期 #001 已结束。是否开始新的诊疗周期？[ Y / N ]"
