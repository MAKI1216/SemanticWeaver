### 任务二：BugFix — 关键词无法拖拽到推演板

**工作流**：🔧 BugFix 快捷路径（工程师定位 + 修复）

**问题描述**：
玩家无法将对话中的高亮关键词拖拽到右侧推演板上，实际上所有拖拽操作都失效了。

**根因定位**：
`js/board.js` 的 `onMouseUp()` 函数中，`cleanupDrag()` 在 `handleDrop()` 之前被调用。`cleanupDrag()` 会把 `dragState.source`、`dragState.sourceData`、`dragState.originalCard` 全部重置为 `null`，导致紧接着的 `handleDrop()` 执行时所有对 `dragState` 的判断全部失败。

**修复内容**：
1. `js/board.js`：交换 `handleDrop()` 和 `cleanupDrag()` 的调用顺序——先处理放置逻辑，再清理拖拽状态
2. `js/board.js`：给 `initGlobalListeners()` 添加 `globalListenersInitialized` 守卫，防止 `newGame()` / `continueGame()` 重复调用时事件监听器被重复绑定

### 任务三：BugFix — Trial 4 合成逻辑炸弹后提交卡住

**工作流**：🔧 BugFix 快捷路径

**问题描述**：
Trial 4（终章）中，玩家成功合成"名为真相的逻辑炸弹"金色卡片后，拖入提交区游戏卡住，没有触发结局画面。

**根因定位**：
Trial 4 中，普通提交区（"将最终线索拖到这里提交"）和特殊目标区（"集团云端数据库"）同时可见。玩家习惯性地把金色卡片拖到普通提交区，触发了 `completeTrial('trial_4')` 流程：
- Trial 4 的 `outro` 是空字符串 → `showNarration("")` → 回调仅设 `gameOver = true`
- 没有触发任何结局画面 → 游戏静默卡死

**修复内容**：
1. `js/game.js` `startTrial()`：Trial 4 中调用 `showSubmitZone(false)` 隐藏普通提交区，只显示特殊目标区
2. `js/game.js` `continueGame()`：存档恢复时同样隐藏普通提交区
3. `js/game.js` `onSubmitCard()`：新增 Trial 4 安全守卫——Trial 4 中非特殊目标提交直接拒绝，提示"需要更强大的目标……将线索拖向集团云端数据库！"
4. `js/game.js` `completeTrial()`：末尾 Trial（无后续章节）兜底处理，不再走空 `outro` 流程，直接触发 `gameOver`

## 2026-06-18（Bug 修复）

### Bug 1：跨对话关键词无法提取

**现象**：进入新对话段后，无法再拖拽上一段对话中的关键词（如 Trial 1 的"很重"在翻到第三段对话后无法提取）。

**根因**：`archiveCurrentDialogue()` 用 `innerHTML` 字符串克隆节点写入历史区时，`data-bound="true"` 属性会被序列化进 HTML 字符串，导致克隆出的 DOM 节点带 `data-bound="true"` 但没有事件监听器。下一段对话打完字后，`bindKeywordEvents()` 扫描所有 `.keyword`，看到历史节点已有 `data-bound="true"` 就跳过，事件永远无法绑定。

**修复方案**（`js/dialogue.js`）：将 `archiveCurrentDialogue()` 从 `innerHTML` 克隆改为**真实 DOM 节点移动**（直接 `appendChild` 子节点）。移动后，子节点上绑定的事件监听器完整保留，无需重新绑定。

**验证**：`node -c` 语法通过，逻辑路径覆盖历史/当前节点移动场景。

**修改文件**：`js/dialogue.js`（仅 `archiveCurrentDialogue` 函数）

### Bug 2：矛盾标记按钮不可见 + 操作流程不清晰

**现象**：玩家无法发现 ⚡ 标记矛盾按钮，无法触发矛盾标记机制。

**根因**：`#mark-contradiction-btn` 默认 `opacity: 0.4`，在深色背景上几乎完全看不见；且没有智能提示引导玩家。

**修复方案**（`js/board.js` + `css/styles.css`）：
1. 按钮默认 opacity 从 0.4 提到 0.7
2. 新增 `updateMarkButtonHint()` 函数——板上有 2+ 张卡片时，⚡ 按钮添加 `hint-available` 类发出金色微脉冲
3. 标记模式视觉强化——推演板红色内阴影，卡片 hover 红色高亮
4. 提示信息更清晰——"⚡ 标记模式已开启：点击两张卡片来标记它们之间的矛盾（再点一次取消选中）"
5. `createCard` / `removeCard` / `clearBoard` / `performReset` 末尾均调用 `updateMarkButtonHint()`

**修改文件**：`js/board.js`、`css/styles.css`

### Bug 3：矛盾连线不跟随卡片移动

**现象**：标记矛盾后两卡片间画了红色虚线连线，但拖动卡片到新位置后，连线仍停留在原坐标，不跟随移动。

**根因**：`showContradictionLine()` 只存了 SVG 元素到 `contradictionLines` 数组，没有记录是哪两张卡片的连线；卡片移动时（`updateCardPosition`）没有任何更新连线坐标的逻辑。

**修复方案**（`js/board.js`）：
1. `contradictionLines` 存储结构从 `[svg]` 改为 `[{svg, line, cardAId, cardBId}]`——每条连线记录关联的两张卡片 ID
2. `showContradictionLine(elA, elB, cardAId, cardBId)` 新增卡片 ID 参数
3. 新增 `updateContradictionLines()` 函数——根据卡片当前 DOM 位置重算所有连线坐标
4. `updateCardPosition()` 末尾调用 `updateContradictionLines()`——卡片移动后连线实时跟随
5. 新增 `removeContradictionLinesForCard(cardId)` 函数——卡片被移除时自动清理关联连线
6. `removeCard()` 中调用 `removeContradictionLinesForCard()`
7. `clearBoard()` / `performReset()` 中调用 `clearContradictionLines()` 清空数组

**修改文件**：`js/board.js`（`showContradictionLine`、`updateCardPosition`、`removeCard`、`clearBoard`、`performReset` + 新增 `updateContradictionLines`、`removeContradictionLinesForCard`）
