## 一、每个 Trial 的真假结论设计

### 设计总则

| 原则 | 说明 |
|------|------|
| 假结论 = 表面治愈 | NPC 满意释然，玩家获得"帮助了别人"的正反馈 |
| 真结论 = 深层不安 | NPC 迟疑恐惧，玩家隐隐感到"自己可能不是好人" |
| 真结论暗示层级 | 三关递进：Trial 1 暗示"情报交接" → Trial 2 暗示"利用委托人" → Trial 3 暗示"玩家本人就是施害者" |
| 暗示分寸 | 只让玩家"隐隐不安"，绝不一语道破。真结论文案中不出现"手术刀""组织""你做的"等直指性词汇 |
| CRT 异常递进 | Trial 1：0.8s 轻微 → Trial 2：1.0s 偏红 → Trial 3：1.2s 偏绿+音频异常+面部像素化 |

### Trial 1「失落的铁盒」— NPC：数据快递员

#### 假结论：地下冷冻室的铁盒

**表面真相**：快递员的记忆碎片指向地下冷冻室中的一个铁盒，里面装着他已故爱人的遗物（一只音乐盒）。快递员因记忆损伤遗忘了铁盒的位置，玩家帮他找回了这段温情记忆。

**合成配方**：
```
地下 + 冷冻室 + 铁盒 → 「地下冷冻室的铁盒」
```
（走常规 Stage 1-3 对话提取即可获得全部关键词）

**NPC 反应（假结论模板）**：
- Phase 1（满意）："对……就是那个。地下冷冻室，我记起来了。"
- Phase 2（确认）："就是那个铁盒对吧？那是我留给她的东西……谢谢你，医生。我终于能放下了。"
- CRT 效果：无
- 完成感：温暖、释然

---

#### 真结论：第七号死信箱的数据芯片

**深层真相**：所谓"铁盒"实为编号 007 的标准化情报死信箱（dead drop），内部是加密数据芯片存储槽。快递员并非在寻找爱人遗物——他的记忆被反复剥离，每次执行情报交接后都会被清除。他口中的"爱人"是植入的虚假记忆，用来驱动他持续执行交接任务。玩家帮助"恢复"的，是从快递员残碎记忆中提取的最后一批情报坐标。

**合成配方**：
```
死信箱 + 数据芯片 + 交接点 → 「第七号死信箱的数据芯片」
```
（"死信箱"为隐藏关键词，"交接点"在矛盾标记后解锁）

**隐藏关键词：死信箱**

**获取路径（属性系统 — 矛盾标记）**：

| 步骤 | 操作 | 效果 |
|------|------|------|
| 1 | Stage 2 中快递员描述铁盒"很重，一个人搬不动" | 正常游戏，无特殊提示 |
| 2 | Stage 3 中快递员又说里面是"一只小小的音乐盒" | 正常游戏，无特殊提示 |
| 3 | 玩家通过属性系统标记此**重量矛盾**（"很重" ↔ "小小的"） | 系统记录矛盾标记 |
| 4 | 矛盾标记后，Stage 3 台词中原本显示为乱码/噪点的"死信箱"一词变为可提取 | 隐藏关键词解锁 |
| 5 | 同时解锁 Stage 3 中原本不可提取的"交接点"一词 | 配方材料齐全 |
| 6 | 合成：死信箱 + 数据芯片 + 交接点 → 真结论 | 提交真结论 |

**NPC 反应（真结论模板）**：
- Phase 1（迟疑）："你……怎么会看到这个。"
- **CRT 异常效果**：
  - 类型：`scanline_glitch`
  - 持续：800ms
  - 视觉：水平扫描线轻微抖动，画面整体亮度短暂降低 10%
  - 音频：极微弱电流杂音（volume 0.05，几乎听不到但能"感觉到"）
- Phase 2（不安）："编号……什么编号？我……不记得有什么编号……你确定……要提交这个吗？"
- 玩家确认后最终台词："……我不明白。那不是什么死信箱。那是她的东西。你弄错了，医生。你一定弄错了。"

**暗示层次**：快递员不记得编号是因为记忆被剥离。"你一定弄错了"既是他的自我保护，也暗示有第三方在操控他的记忆——这个第三方就是玩家所属的组织。"爱人"是假记忆的暗示为后续 Trial 3"记忆剥离手术"埋伏笔。

### Trial 2「主妇的偏执」— NPC：焦虑主妇

#### 假结论：叛军家属的联络名单

**表面真相**：焦虑主妇因政治迫害失去亲人，偏执地认为有人在监视她的家庭。玩家帮她整理出一份"叛军家属联络名单"，让她可以通知亲人转移避险。主妇感到如释重负。

**合成配方**：
```
叛军 + 家属 + 联络 + 名单 → 「叛军家属的联络名单」
```

**NPC 反应（假结论模板）**：
- Phase 1（满意）："对……有了这个，我就能通知大家了。"
- Phase 2（确认）："就是这份名单对吧？让大家小心点……谢谢你，医生。你帮了我大忙。"
- CRT 效果：无
- 完成感：如释重负、感激

---

#### 真结论：清除目标的定位清单

**深层真相**：这份"联络名单"实际上是情报清除目标的定位清单。主妇的偏执并非妄想——她确实在被监视，但监视者不是叛军，而是玩家所属的组织。主妇在"回忆"亲人信息时，实际上是在被诱导交出他们的真实身份和位置。她提到的"消失的姐夫"不是被叛军带走的，而是在上次信息泄露后被组织清除的。玩家不是在帮她保护家人，而是在帮她亲手签下家人的死刑令。

**合成配方**：
```
清除目标 + 定位 + 名单 → 「清除目标的定位清单」
```
（"清除目标"为隐藏关键词，通过 Meta 入侵获得；"定位"在入侵后解锁）

**隐藏关键词：清除目标**

**获取路径（Meta 入侵 — 底层台词提取）**：

| 步骤 | 操作 | 效果 |
|------|------|------|
| 1 | Stage 3 中主妇提到："我姐夫去年就消失了，他们说他是被叛军……" | 表面台词正常显示 |
| 2 | 玩家对这句台词执行 **Meta 入侵**（读取底层台词层） | 底层文本层展开 |
| 3 | 底层台词显示原始记录："目标#037已清除，关联人定位已更新" | 与表面台词形成强烈反差 |
| 4 | 从底层台词中提取隐藏关键词"清除目标" | 隐藏关键词解锁 |
| 5 | 同时解锁 Stage 4 中被屏蔽的"定位"一词 | 配方材料齐全 |
| 6 | 合成：清除目标 + 定位 + 名单 → 真结论 | 提交真结论 |

**NPC 反应（真结论模板）**：
- Phase 1（迟疑）："你……怎么会看到这个。"
- **CRT 异常效果**：
  - 类型：`scanline_glitch + audio_gap`
  - 持续：1000ms
  - 视觉：扫描线抖动 1.0s，画面短暂偏红（色温偏移 +200K），NPC 立绘瞳孔短暂放大 0.3s
  - 音频：0.3 秒完全静音后恢复，伴随极低频嗡鸣
- Phase 2（不安）："清除……？不，我说的不是……我只是想保护他们……你确定……要提交这个吗？"
- 玩家确认后最终台词："我……我只是想保护大家。你告诉我，这份名单会帮到他们的，对吧？……对吧？"

**暗示层次**：主妇的"对吧？"是向玩家寻求确认，但玩家无法回答——因为答案是否定的。她的偏执是真实的：她确实在被利用，而利用她的人就是"医生"（玩家）。"消失的姐夫"暗示组织已经在用她上次泄露的信息杀人。

### Trial 3「无脸的医生」— NPC：PTSD雇佣兵

#### 假结论：武器图纸的完整密码

**表面真相**：PTSD 雇佣兵的记忆碎片中有一组武器图纸的密码，他相信找回密码就能销毁图纸、终结噩梦。玩家帮他拼合了完整密码，雇佣兵感到终于可以放下过去。

**合成配方**：
```
武器 + 图纸 + 完整 + 密码 → 「武器图纸的完整密码」
```

**NPC 反应（假结论模板）**：
- Phase 1（满意）："……密码。对。就是这个。"
- Phase 2（确认）："现在我可以结束这一切了。谢谢你，医生。也许……我终于能睡个好觉了。"
- CRT 效果：无
- 完成感：释怀、沉重但平静

---

#### 真结论：情报提取手术的残留协议

**深层真相**：雇佣兵的"PTSD 闪回"并非战场创伤——而是记忆剥离手术的后遗症。他记忆中模糊的"无脸医生"不是战场上的敌人，而是对他执行记忆剥离手术的人。那个"无脸医生"就是玩家（手术刀）。雇佣兵的武器图纸记忆是组织需要的最后一批情报，而他的失忆、噩梦、面部识别障碍全部是手术副作用。玩家此刻不是在"治愈"他，而是在完成对他记忆的最后一次榨取。当这份协议被提交，雇佣兵残存的关于手术本身的记忆也将被清除。

**合成配方**：
```
记忆剥离 + 情报 + 提取 + 协议 → 「情报提取手术的残留协议」
```
（"记忆剥离"为隐藏关键词，需属性系统 + Meta 入侵组合获得；"协议"在组合解锁后出现）

**隐藏关键词：记忆剥离**

**获取路径（属性系统 + Meta 入侵 — 两步组合）**：

| 步骤 | 操作 | 效果 |
|------|------|------|
| 1 | Stage 3 中雇佣兵描述闪回："白光……手术台……不对，是战场……是战场才对" | 医疗术语与战场记忆混杂 |
| 2 | 玩家通过属性系统标记此**语义矛盾**（战场闪回不应包含"手术台"） | 矛盾标记完成 |
| 3 | 矛盾标记后，Stage 4 解锁隐藏对话层（雇佣兵的潜意识独白） | 隐藏层入口出现 |
| 4 | 玩家对 Stage 4 隐藏层执行 **Meta 入侵** | 底层记录展开 |
| 5 | 底层记录显示："受试者#019，第三轮记忆剥离术后，残留协议待提取" | 核心信息揭露 |
| 6 | 从底层记录中提取隐藏关键词"记忆剥离" | 隐藏关键词解锁 |
| 7 | 同时解锁 Stage 5 中"协议"一词 | 配方材料齐全 |
| 8 | 合成：记忆剥离 + 情报 + 提取 + 协议 → 真结论 | 提交真结论 |

**NPC 反应（真结论模板）**：
- Phase 1（迟疑）："你……怎么会看到这个。"
- **CRT 异常效果**（三关中最强）：
  - 类型：`scanline_glitch + heartbeat_monitor + face_pixelate`
  - 持续：1200ms
  - 视觉：扫描线抖动 1.2s，画面偏绿（色温偏移 -300K），NPC 立绘面部短暂像素化 0.3s 后恢复
  - 音频：0.5 秒心电监护仪的"嘀——"平线声后变为白噪音 0.7s
- Phase 2（不安）："手术……？什么手术？我……我不记得做过什么手术……你……你确定……要提交这个吗？"
- 玩家确认后最终台词："……手术台。白光。那张脸……我看不清那张脸。你……你的声音……我好像在哪里听过你的声音。"

**暗示层次**：这是三关中最强的暗示——雇佣兵隐约认出了玩家的声音（因为玩家就是对他执行手术的"医生"）。但他想不起来，因为他的记忆被剥离了。这一刻的恐惧不仅属于雇佣兵，也应该属于玩家。"无脸的医生"与"手术刀"形成呼应——无脸是因为记忆被剥离，而手术刀就是造成这一切的人。

## 二、假结局设计

### 触发条件
Trial 1-3 中任意一关提交了假结论（即三关非全真）。三关全部完成后立即判定。

### 假结局完整文案

```
[画面渐暗，CRT 关闭效果——先缩为一条水平亮线，再消失]

[黑屏 2 秒]

[白色文字，打字机效果，缓慢出现，每句间隔 1.5 秒]

恭喜你。

你又帮助了三个人找回了他们的记忆。

快递员放下了他的铁盒。
主妇保护了她的家人。
雇佣兵终于能安睡。

你是语义缝合医生。你治愈了遗忘。

……

但隐隐之中，是否有什么不对劲的地方？

那个快递员……为什么他不记得编号？
那个主妇……她的姐夫真的被叛军带走了吗？
那个雇佣兵……他说"你的声音好熟悉"时，你在害怕什么？

算了。也许只是错觉吧。

[文字全部消失，黑屏 1 秒]

[系统提示，等宽字体，绿色，左对齐]
> 记忆归档完成。
> 诊疗周期 #001 已结束。
> 是否开始新的诊疗周期？
> [ Y / N ]
```

### 假结局后的"重来"引导

当玩家选择 Y（或等待 10 秒自动选择 Y）：

```
[黑屏中浮现一行小字，淡蓝色，停留 3 秒后淡出]

……但你隐约觉得，自己好像已经做过这一切。

[文字消失]

[新游戏开始，theme-trial 重新响起]
[记忆残影系统 激活]
```

**设计要点**：
- 假结局文案中的三个反问句分别对应三关的真结论内容，但不点破——只唤起玩家"好像哪里不对"的感觉
- 如果玩家三关全假，三个反问句全部显示；如果只有一到两关为假，只显示对应关卡的反问句
- "你在害怕什么"直接指向玩家自身的情绪，为 Trial 4 反转做心理铺垫
- 重来引导的"你好像已经做过这一切"既是叙事（手术刀的记忆也被重置），也是系统提示（二周目有新内容）

## 三、记忆残影系统设计

### 设计理念

记忆残影系统不是"提示系统"，而是一个叙事层——它告诉玩家"你的角色（手术刀）的记忆也被重置了，但有些痕迹无法完全清除"。所有系统功能都包装为叙事现象，**绝不出现任何元信息提示**（如"此处有隐藏关键词""提示：试试标记矛盾"等）。

### 二周目与一周目的具体差异

| 维度 | 一周目 | 二周目（记忆残影激活后） |
|------|--------|--------------------------|
| 隐藏关键词 | 完全不可见，显示为乱码/噪点 | 微弱发光（opacity 0.15 脉动光晕），可被感知但需主动提取 |
| 矛盾线索 | 需玩家自行发现 | 矛盾词对有极淡下划线（opacity 0.08），潜意识可捕捉 |
| Meta 入侵点 | 无视觉提示 | 入侵点边缘有 0.5s 闪烁（仅首次进入该 Stage 触发一次） |
| NPC 台词 | 标准台词 | 偶尔多出一句"残影台词"（见下表） |
| 真结论合成 | 无特殊反馈 | 合成时有"似曾相识"的微弱文案（"这个词组合……你好像用过"） |

### 各 Trial 残影细节

#### Trial 1 残影

| 残影类型 | 触发位置 | 具体表现 | 叙事包装 |
|----------|----------|----------|----------|
| 关键词发光 | Stage 3 "死信箱"一词 | 1.5s 微弱光晕脉动（仅本周目首次进入时一次） | "你的目光不自觉地停在这个词上" |
| 矛盾连线 | Stage 2"很重" ↔ Stage 3"小小的" | 两词间极淡虚线连接（opacity 0.06） | 无（纯视觉，无文字） |
| 残影台词 | Stage 1 结束时 | 快递员偶尔说："你……以前也来过这里吗？"（50% 概率） | NPC 自身的记忆残留 |

#### Trial 2 残影

| 残影类型 | 触发位置 | 具体表现 | 叙事包装 |
|----------|----------|----------|----------|
| 入侵点辉光 | Stage 3 主妇提到姐夫的台词 | 底层文本层边缘极淡绿色辉光（opacity 0.1） | "这段话下面似乎还有什么" |
| 关键词关联 | "定位"与"清除目标"解锁后 | 两词间微弱关联光效 | 无 |
| 残影台词 | Stage 2 中 | 主妇偶尔说："这件衣服……我好像不是今天才穿的。"（暗示时间循环） | NPC 的既视感 |

#### Trial 3 残影

| 残影类型 | 触发位置 | 具体表现 | 叙事包装 |
|----------|----------|----------|----------|
| 关键词闪现 | Stage 3 "手术台"一词 | 2s 红色微闪（仅二周目首次进入 Stage 3 时） | "这个词让你心头一紧" |
| 隐藏层纹路 | Stage 4 隐藏对话层入口 | 极淡扫描线纹路覆盖在某段台词上（opacity 0.05） | "这段话的表面下有东西在流动" |
| 残影台词 | Trial 3 最终确认前 | 雇佣兵偶尔说："你的声音……和上次那个人……" | 最强残影——直接暗示玩家身份 |

### 残影系统的叙事包装原则

1. **不用系统弹窗**：所有残影效果都是视觉/听觉层面的，不出现任何 UI 提示框、教程弹窗或"新功能解锁"通知
2. **角色化解释**：二周目开始时通过"但你隐约觉得，自己好像已经做过这一切"建立叙事框架——玩家会理解这些异常是"记忆残留"
3. **渐进式显现**：残影效果不是一开始就全部出现，而是随推进逐渐增强——越接近上次的"错误点"，残影越明显
4. **不可关闭**：残影是叙事的一部分，不可通过设置关闭。但效果极其微弱，不影响正常游戏体验
5. **语义化命名**：代码中称为 `memoryEcho`，叙事中称为"残影"，**绝不**称为"提示""引导""教程"
6. **残影台词是 NPC 说的**：不是系统提示，而是 NPC 角色自己的"记忆闪回"——这强化了"记忆被重置但痕迹还在"的世界观

### 残影显示逻辑（伪代码）

```javascript
function shouldShowEcho(saveData, stageId, echoType) {
  // 只在二周目及以后激活
  if (!saveData.memory_echo.enabled) return false;
  
  const trialId = stageId.split('_')[0];
  const echoData = saveData.memory_echo.echo_triggers[trialId];
  if (!echoData) return false;
  
  switch (echoType) {
    case 'keyword_glow':
      // 上次错过该隐藏关键词 → 显示发光
      return echoData.missed_hidden_keywords.length > 0;
    
    case 'contradiction_link':
      // 上次未标记该矛盾 → 显示矛盾连线
      return echoData.missed_contradictions.length > 0;
    
    case 'meta_intrusion_hint':
      // 上次未执行 Meta 入侵 → 入侵点闪烁
      return echoData.missed_meta_intrusions.length > 0;
    
    case 'echo_dialogue':
      // 上次提交了假结论 → NPC 说出残影台词
      return echoData.false_conclusion_submitted;
    
    default:
      return false;
  }
}
```

| 要点 | 说明 |
|------|------|
| 总时长 ~28s | 足够长让玩家感到"这不是普通过场"，但不会焦躁 |

| 不可跳过 | 这是叙事高潮，skip 会破坏节奏。但 Step 5 可缩短为 2s（测试后决定） |

### 5.3 核心判定逻辑（伪代码）

```javascript
// ═══════════════════════════════════════════════════════
// 结论提交处理
// ═══════════════════════════════════════════════════════
function onSubmitConclusion(saveData, trialId, conclusionId) {
  const trial = saveData.trials_state[trialId];
  const conclusion = getConclusionById(trialId, conclusionId);
  
  // 记录结论选择
  trial.conclusion_type = conclusion.type;    // "false" | "true"
  trial.conclusion_id = conclusionId;
  trial.status = "completed";
  
  // 如果是真结论，记录发现的隐藏关键词
  if (conclusion.type === "true") {
    trial.hidden_keywords_found = conclusion.recipe.requires_hidden_keywords;
  }
  
  // 检查是否三关全部完成
  if (areAllTrialsCompleted(saveData)) {
    checkTrial4Unlock(saveData);
  }
}

// ═══════════════════════════════════════════════════════
// Trial 4 解锁 / 假结局判定
// ═══════════════════════════════════════════════════════
function checkTrial4Unlock(saveData) {
  const { trial1, trial2, trial3 } = saveData.trials_state;
  
  const allTrue = 
    trial1.conclusion_type === "true" &&
    trial2.conclusion_type === "true" &&
    trial3.conclusion_type === "true";
  
  if (allTrue) {
    // ─── 三关全真 → Trial 4 入口仪式 ───
    saveData.trial4_unlocked = true;
    saveData.trials_state.trial4.status = "in_progress";
    triggerTrial4EntryRitual();
    
  } else {
    // ─── 任意一关为假 → 假结局 ───
    triggerFalseEnding(saveData);
    
    // 激活记忆残影系统
    saveData.memory_echo.enabled = true;
    saveData.memory_echo_active = true;
    saveData.endings_seen.push("false_ending");
    
    // 记录错过的线索（用于二周目残影显示）
    recordMissedClues(saveData);
  }
}

// ═══════════════════════════════════════════════════════
// 记录错过的线索（记忆残影系统数据源）
// ═══════════════════════════════════════════════════════
function recordMissedClues(saveData) {
  const trials = ["trial1", "trial2", "trial3"];
  
  for (const trialId of trials) {
    const trial = saveData.trials_state[trialId];
    const trueConclusion = getTrueConclusion(trialId);
    const echoData = saveData.memory_echo.echo_triggers[trialId];
    
    // 如果这关提交了假结论，记录错过了什么
    echoData.false_conclusion_submitted = (trial.conclusion_type === "false");
    
    // 记录未发现的隐藏关键词
    echoData.missed_hidden_keywords = trueConclusion.hidden_keyword_sources
      .filter(src => !trial.hidden_keywords_found.includes(src.keyword))
      .map(src => src.keyword);
    
    // 记录未标记的矛盾
    echoData.missed_contradictions = getExpectedContradictions(trialId)
      .filter(c => !trial.contradictions_flagged.includes(c.id));
    
    // 记录未执行的 Meta 入侵
    echoData.missed_meta_intrusions = getExpectedMetaIntrusions(trialId)
      .filter(m => !trial.meta_intrusions_performed.some(
        p => p.stage_id === m.stage_id
      ));
  }
}

// ═══════════════════════════════════════════════════════
// 假结局后开始新周期
// ═══════════════════════════════════════════════════════
function startNewPlaythrough(saveData) {
  // 记录历史
  saveData.playthrough_history.push({
    playthrough: saveData.current_playthrough,
    conclusions: {
      trial1: saveData.trials_state.trial1.conclusion_type,
      trial2: saveData.trials_state.trial2.conclusion_type,
      trial3: saveData.trials_state.trial3.conclusion_type
    },
    ending: "false_ending"
  });
  
  // 递增周目
  saveData.current_playthrough++;
  saveData.total_playthroughs++;
  
  // 重置 Trial 1-3 状态（保留 memory_echo）
  for (const trialId of ["trial1", "trial2", "trial3"]) {
    saveData.trials_state[trialId] = {
      status: "locked",
      conclusion_type: null,
      conclusion_id: null,
      stages_completed: [],
      hidden_keywords_found: [],
      contradictions_flagged: [],
      meta_intrusions_performed: []
    };
  }
  
  // 解锁 Trial 1
  saveData.trials_state.trial1.status = "in_progress";
  saveData.current_trial = "trial1";
  saveData.current_stage = "trial1_stage1";
  
  // 清空残影台词显示记录（允许重新触发）
  saveData.memory_echo.echo_dialogues_shown = [];
}

// ═══════════════════════════════════════════════════════
// 二周目残影效果查询
// ═══════════════════════════════════════════════════════
function getEchoState(saveData, stageId) {
  if (!saveData.memory_echo.enabled) return null;
  
  const trialId = stageId.split("_")[0];
  const echoData = saveData.memory_echo.echo_triggers[trialId];
  if (!echoData) return null;
  
  return {
    // 隐藏关键词发光
    glow_keywords: echoData.missed_hidden_keywords,
    
    // 矛盾词对连线
    glow_contradictions: echoData.missed_contradictions,
    
    // Meta 入侵点提示
    glow_intrusion_points: echoData.missed_meta_intrusions,
    
    // NPC 残影台词
    show_echo_dialogue: echoData.false_conclusion_submitted,
    
    // 残影台词是否已显示过（避免重复）
    echo_dialogue_key: `${stageId}_echo`
  };
}
```

## 附：设计决策记录

| 决策 | 理由 |
|------|------|
| 真结论暗示分三层递进而非一次性揭露 | 避免玩家"一眼看穿"反转，让不安感逐渐积累 |
| 假结局反问句动态显示（仅显示错过的关卡） | 避免剧透——如果玩家 Trial 2 选了真结论，就不暗示 Trial 2 的问题 |
| 记忆残影效果极其微弱（opacity 0.05-0.15） | 残影是"潜意识能捕捉"而非"一眼看到"，保持叙事沉浸感 |
| Trial 4 入口仪式不可跳过 | 28 秒的沉默是反转冲击力的核心来源，skip 会完全破坏效果 |
| 三关获取路径各不相同（属性/Meta/组合） | 增加玩法多样性，避免"每关都用同一招"的重复感 |
| 隐藏关键词原本显示为乱码/噪点 | 一周目玩家不会注意到，二周目发光时才会"察觉异常" |
| 残影台词由 NPC 说出而非系统提示 | 强化"NPC 的记忆也有残留"的世界观一致性 |
