## Bug 修复：跨对话关键词无法提取

- **根因**：`archiveCurrentDialogue()` 用 `innerHTML` 字符串克隆节点，`data-bound="true"` 被序列化进 HTML 字符串。克隆节点带 `data-bound` 但无事件监听器，`bindKeywordEvents` 看到 `data-bound` 就跳过，历史对话关键词永远无法提取。
- **修复**：`archiveCurrentDialogue()` 改为真实 DOM 节点移动（`appendChild` 子节点）。事件监听器随节点完整保留，无需重绑。
- **影响文件**：`js/dialogue.js`（仅 `archiveCurrentDialogue` 函数）

## Bug 修复：矛盾连线不跟随卡片移动

- **问题**：标记矛盾后画了红色虚线连线，但拖动卡片后连线不跟随移动，停留在原坐标
- **根因**：`showContradictionLine` 只存 SVG 元素到数组，没存卡片 ID 关联；`updateCardPosition` 没有更新连线坐标的逻辑
- **修复**（`js/board.js`）：
  1. `contradictionLines` 存储从 `[svg]` 改为 `[{svg, line, cardAId, cardBId}]`
  2. `showContradictionLine` 新增 cardAId/cardBId 参数
  3. 新增 `updateContradictionLines()` —— 根据卡片当前 DOM 位置重算连线坐标
  4. `updateCardPosition` 末尾调用 `updateContradictionLines()`
  5. 新增 `removeContradictionLinesForCard(cardId)` —— 卡片移除时清理关联连线
  6. `clearBoard`/`performReset` 中调用 `clearContradictionLines()` 清空数组

## Bug 修复：真结论产物无法提交（核心 Bug）

- **问题**：合成真结论（如 `第七号死信箱的数据芯片`）后无法拖入提交区，而假结论（`废弃诊所的后巷`/`地下冷冻室的铁盒`）可以正常提交
- **根因链（三层嵌套）**：
  1. `board.js:84-87` — `isGolden` 只在 `text === required_submit` 时为 true
  2. Trial1 Stage3 的 `required_submit = "地下冷冻室的铁盒"`（= 假结论），≠ 真结论 `"第七号死信箱的数据芯片"`
  3. **`board.js:752`** — submit zone 检测条件要求 `dragState.sourceData.isGolden === true` → 真结论卡 isGolden=false → **整个 submit zone 检测被跳过** → 无法触发 onSubmitCallback → 真结论流程永远无法触发
- **影响范围**：Trial 1/2/3 的真结论全部无法提交（结构完全相同）
- **修复方案（3 处改动）**：
  1. `board.js`: 新增 `submittableCards[]` 数组，扩展 `setRequiredSubmit(text, extraSubmittable)` 接受第二参数
  2. `board.js:84-87`: `isGolden` 判断从 `text === requiredSubmit` 改为 `submittableCards.indexOf(text) !== -1`
  3. `board.js:752`: submit zone 条件从仅 `isGolden` 改为 `isGolden || submittableCards.contains(cardText)`
  4. `game.js:933`: 设置 Stage 时提取 `conclusions[].label` 和 `conclusions[].recipe.result` 作为额外可提交列表传入 `setRequiredSubmit()`
- **修改文件**：`js/board.js` + `js/game.js`
- **语法验证**：两个文件均 `node --check` 通过

### Trial 4 入口条件 Bug 修复（conclusion_type 丢失）

- **修复 2**（`continueGame()`）：从 `saveData.trials_state` 读取三个 Trial 的 `conclusion_type` 到 `state.conclusionTypes`
- **修复 3**（`completeTrial()`）：Trial 3 完成时重新从 localStorage 读取所有三个 Trial 的 `conclusion_type`（防止内存丢失）
